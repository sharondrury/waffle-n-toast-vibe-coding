Feature: Phone Icon Vibrate Animation
---------------------------------------
What was implemented:
  - Pure CSS @keyframes phone-ring on the SVG element
  - Rotates left/right in a rapid burst then pauses (2.4s loop)
  - Mimics a phone vibrating on a table: 8 quick alternating rotations, then rest
  - transform-origin: center for natural pivot

Update — Phone icon changed to Font Awesome:
  - Replaced 📱 emoji with <i className="fa-solid fa-mobile-button"></i>
  - Installed @fortawesome/fontawesome-free via npm
  - Imported '@fortawesome/fontawesome-free/css/all.min.css' in src/main.jsx
  - .nav__phone-icon wrapper still applies the phone-ring animation as before
  - icon colour inherits from CSS (currentColor / parent colour)
Files changed:
  src/website/components/navBar/navBar.jsx
  src/main.jsx

Feature: Nav Links Always Centred
----------------------------------
What was implemented:
  - .nav given position: relative so children can use absolute positioning
  - .nav__links given position: absolute; left: 50%; transform: translateX(-50%)
  - This pins the links pill to the true centre of the nav at all viewport widths,
    independent of the logo and phone button widths on either side
  - justify-content: space-between kept on .nav so logo stays left and phone stays right
Files changed:
  src/website/components/navBar/navBar.scss

Learning note:
  NavLink from react-router-dom automatically adds an `active` class when the
  current URL matches the `to` prop. We leverage this to style the active link
  differently without any manual state tracking.
