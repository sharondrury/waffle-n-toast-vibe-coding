Feat-005 | Scroll-Driven Direction + Animation Loop
-----------------------------------------------------
File: vibe-coding/src/website/pages/homePage/companyLogo/companyLogo.jsx

Full useEffect block:

  const SPEED = 1

  useEffect(() => {
    function handleScroll() {
      const currentY = window.scrollY
      targetDirectionRef.current = currentY > lastScrollY.current ? 1 : -1
      lastScrollY.current = currentY
    }

    function animate() {
      if (!trackRef.current) return

      // Lerp smooth direction toward target
      smoothDirectionRef.current +=
        (targetDirectionRef.current - smoothDirectionRef.current) * 0.08

      positionRef.current += SPEED * smoothDirectionRef.current

      const halfWidth = trackRef.current.scrollWidth / 2

      // Loop bounds
      if (positionRef.current >= halfWidth) positionRef.current = 0
      if (positionRef.current < 0) positionRef.current = halfWidth - 1

      trackRef.current.style.transform = `translateX(-${positionRef.current}px)`
      animFrameRef.current = requestAnimationFrame(animate)
    }

    window.addEventListener('scroll', handleScroll, { passive: true })
    animFrameRef.current = requestAnimationFrame(animate)

    return () => {
      window.removeEventListener('scroll', handleScroll)
      cancelAnimationFrame(animFrameRef.current)
    }
  }, [])

Notes:
  - { passive: true } on the scroll listener improves scroll performance.
  - All values in refs — no setState inside the animation loop.
  - The loop starts immediately on mount so the carousel is always moving
    even before the user scrolls. This gives a live feel on page load.
    Initial direction is left-to-right (directionRef default = 1).

NOTES & AFTERTHOUGHTS
======================

1. Similarity to showcase.jsx
   ----------------------------
   This component intentionally mirrors the pattern in showcase.jsx (EPIC-005).
   If showcase.jsx already has a working loop/animation approach, Claude Code
   should reference it for consistency — particularly the loop reset logic and
   ref-based animation. The key difference here is scroll-driven direction rather
   than a constant auto-scroll.

2. Initial scroll direction on page load
   ----------------------------------------
   On first load, smoothDirectionRef starts at 1 (left to right).
   The carousel moves left to right until the user scrolls.
   This is intentional — it gives the component life immediately.
   If the preference is for it to be still until first scroll, wrap the
   animate() call start in a flag that only begins after the first scroll event.

3. Performance — will-change
   ---------------------------
   will-change: transform on .company-logo__track hints to the browser to
   promote the element to its own GPU layer. This prevents the animation from
   causing repaints on the rest of the page. Keep this in the SCSS.

4. Reduced motion accessibility
   --------------------------------
   Consider adding a prefers-reduced-motion media query to pause the animation:
     @media (prefers-reduced-motion: reduce) {
       .company-logo__track { animation: none; }
     }
   For the rAF-based approach, check matchMedia in the useEffect and skip the
   animate() call if reduced motion is preferred. Nice-to-have for this epic,
   not a blocker.
