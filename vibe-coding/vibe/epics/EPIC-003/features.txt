Amend-004 | Heading sticky — viewport-centred scroll follow, stops 50px from section bottom
  - As the user scrolls through the companyServices section, the heading should
    travel with them and remain visible — centred vertically in the viewport.
  - Use CSS position: sticky with top: calc(50vh - [half heading height]) so
    the heading naturally sits in the vertical centre of the viewport while sticky.
    Estimated heading height is ~100px, so top: calc(50vh - 50px) is the starting
    point — adjust if the heading wraps differently.
  - The heading must stop 50px before the bottom of the section.
    Keep padding-bottom: 50px on the left column — sticky cannot travel beyond
    its containing block, so this naturally creates the 50px stop buffer.
  - Do NOT use overflow: hidden on the left column (breaks sticky).
  - Amends: Feat-006
  Status: done

Amend-005 | Cards — scroll-position-driven bidirectional animation (replaces Feat-008)
  - Replace the current IntersectionObserver one-shot animation with a
    scroll-event-driven approach so the animation plays forward on scroll down
    and reverses on scroll up.
  - Remove: IntersectionObserver, .is-visible class, CSS transition on cards,
    staggered setTimeout delays.
  - New approach:
      - In useEffect, attach a scroll event listener to window (or use
        requestAnimationFrame for smoothness).
      - For each card, use getBoundingClientRect() to calculate how far into the
        viewport the card has scrolled.
      - Derive a progress value from 0 (card off-screen right) to 1 (card fully
        in view).
        Suggested calculation: clamp((viewportBottom - cardTop) / triggerDistance, 0, 1)
        where triggerDistance is e.g. 300px.
      - Apply opacity and translateX as inline styles driven by progress:
          opacity: progress
          transform: translateX((1 - progress) * 60px)
      - No stagger delay — the natural difference in card positions creates the
        sequential feel as the user scrolls.
      - Clean up the scroll listener on unmount.
  - Replaces: Feat-008
  Status: done

Amend-006 | Icon — top-left alignment, level with heading
  - The icon should align to the top of the card, sitting level with the first
    line of the heading — not vertically centred in the card.
  - Change align-items on .company-services__card from center to flex-start.
  - No other card layout changes.
  - Amends: Feat-007
