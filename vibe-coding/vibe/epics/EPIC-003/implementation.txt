Feature: Feat-008 | Scroll-triggered entrance animation
---------------------------------------------------------
What was implemented:
  Each card starts at opacity 0, translateX(60px). IntersectionObserver in
  useEffect watches each card via cardRefs. On intersection (threshold 0.2),
  adds .is-visible class after a staggered setTimeout delay (i * 150ms).
  Observer disconnects immediately so the animation never re-triggers.
  CSS transition: opacity 0.5s ease, transform 0.5s ease.
Files changed:
  vibe-coding/src/website/pages/homePage/companyServices/companyServices.jsx
  vibe-coding/src/website/pages/homePage/companyServices/companyServices.scss
Engine components considered:
  - timer.jsx — reviewed; not used. IntersectionObserver handles the timing
    natively without needing the engine timer utility.
Note: Once .is-visible is applied it is never removed, so cards stay in
position on scroll back. Observer is disconnected in the same callback to
prevent any re-trigger.
Tradeoffs / follow-up:
  hero.jsx had a pre-existing lint error (react-hooks/set-state-in-effect)
  which blocked `npm run lint`. Fixed by wrapping the two direct setState calls
  in setTimeout(fn, 0) so they run in a callback rather than synchronously in
  the effect body. Behaviour is identical.

Amendment: Amend-005 | Cards — scroll-driven bidirectional animation
---------------------------------------------------------------------
What was implemented:
  Replaced IntersectionObserver (one-shot, Feat-008) with a scroll-event-driven
  approach. A single updateCards function runs on mount and on every scroll event.
  For each card, getBoundingClientRect().top is used to derive a progress value
  (0 = off-screen, 1 = fully in view) using TRIGGER_DISTANCE = 300px.
  Opacity and translateX are applied as inline styles directly on the DOM node,
  avoiding React re-renders on every scroll tick.
  Scrolling up reverses the animation naturally since progress decreases as
  cards move back below the trigger zone.
Files changed:
  vibe-coding/src/website/pages/homePage/companyServices/companyServices.jsx
  vibe-coding/src/website/pages/homePage/companyServices/companyServices.scss
Engine components considered:
  - timer.jsx — not needed; scroll listener handles timing.
Note: opacity: 0 / translateX initial state is now set by the scroll handler
on mount (before first scroll), not by CSS. CSS only has will-change as a
GPU compositing hint.
Tradeoffs / follow-up:
  Stagger delay from Feat-008 removed intentionally — card position offsets
  produce natural sequential reveal as the user scrolls.
