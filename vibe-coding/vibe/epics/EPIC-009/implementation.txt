Feature: Feat-002 | Navbar — hamburger menu
---------------------------------------------
What was implemented:
  - slidePanel.jsx engine component does not exist in this project.
    A lightweight custom panel was built local to navBar.
  - Added menuOpen state to navBar.jsx (alongside existing modalOpen + scrolled).
  - nav__right wrapper div groups the hamburger and phone pill so they stay
    paired on the right at all viewport widths.
  - nav__hamburger button: fa-solid fa-bars icon, hidden at desktop via CSS,
    shown at ≤1000px via display: flex.
  - nav__panel: position fixed, slides in from right via translateX(100%) →
    translateX(0), CSS transition 0.3s ease. Contains About / Projects / Pricing
    NavLinks and a × close button (fa-solid fa-xmark).
  - nav__backdrop: fixed full-screen dark overlay behind the panel.
    Click closes the menu.
  - Each NavLink inside the panel calls closeMenu() onClick so it closes on
    navigation.
  - nav__links (desktop pill) hidden at ≤1000px via display: none.
Files changed:
  src/website/components/navBar/navBar.jsx
  src/website/components/navBar/navBar.scss
Engine components considered:
  - slidePanel.jsx — does not exist in this project. Custom panel built instead,
    kept local to navBar as it has no other consumers.

Learning Notes
---------------
  - position: sticky must be explicitly overridden with position: static inside
    a media query — it does not automatically reset when layout changes.
  - animation: none !important is the safest way to kill JS-toggled CSS
    animations at mobile without changing any JavaScript logic.
  - Slide panel built local to navBar — no engine component needed because it
    has exactly one consumer and is tightly coupled to nav's menu state.
  - var(--mobile-gutter) defined once, consumed everywhere — future gutter
    changes are a one-line edit in colours.scss.
