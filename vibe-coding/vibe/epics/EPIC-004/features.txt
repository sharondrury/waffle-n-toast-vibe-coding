Feat-005 | Accordion pills — layout and open/close behaviour
  - Render by mapping over faqData

      - Gap between pills: 16px
      - Closed state:
          Background: --green-light at 50% opacity (rgba or color-mix)
          Question text: left aligned, 20px → 1.25rem, bold, black
          + icon: right aligned, changes to × when pill is open
      - Open state:
          Question pill background: --green-light at 50% opacity (unchanged)
          Answer revealed below question row:
            Background: --green-light at 80% opacity
            Font size: 18px → 1.125rem
            Smooth expand/collapse transition (max-height or grid-rows technique)
  - Only one pill open at a time:
      Track openIndex in useState (null = all closed)
      Clicking an open pill closes it (set to null)
      Clicking a different pill closes the current one and opens the new one
  Status: done

  Status: done

Feat-007 | Floating question mark decorations

    of the accordion content area

  - They sit outside the 350px max-width constraint of the heading/accordion
  - Colours: pull from existing tokens in vibe-coding/src/assets/scss/ui/colours.scss
      Suggested mapping (confirm tokens available):
        Left top:    var(--green-light)
        Left bottom: var(--green-dark)
        Right top:   var(--right-color)
        Right bottom: var(--bottom-color)
  - Font size: make them large and decorative (e.g. 4rem–6rem), semi-transparent
  - Each '?' has its own unique @keyframes float animation:
      Small translateY and translateX movement within a limited area
      Each keyframe set has slightly different timing, distance and easing
      so all four move independently and never look synchronised
  - Animation durations: stagger between ~3s and ~6s, all set to infinite
  - Should not interfere with page layout (pointer-events: none, user-select: none)
