EPIC 004 — FAQ Section: Implementation Log
==========================================

Implementation notes for Claude Code:
  - Actual folder path is src/website/pages/homePage/faq/ (camelCase) —
    features.txt lists it as homepage/ (lowercase). Always check the real
    project tree first.
  - The section element must use position: relative and be full-width with NO
    max-width. Only the inner .faq__inner wrapper is constrained to 350px.
    This is what allows the absolute-positioned ? decorations to break outside
    the content boundary. If max-width is placed on the section itself the
    decorations will be clipped inside it.
  - Feat-006 animation: use the CSS grid-template-rows technique
    (grid-template-rows: 0fr → 1fr on the answer wrapper, inner div with
    overflow: hidden and min-height: 0). Do NOT use max-height animation —
    it requires an arbitrary large value and causes sluggish ease-out. Do NOT
    use display: none/block — it cannot be transitioned.
  - Add --black: #000000 to colours.scss before using it in faq.scss (no
    hard-coded hex values in component files per rules.txt).
  - color-mix() for 50% and 80% opacity green backgrounds:
      color-mix(in srgb, var(--green-light) 50%, transparent)
      color-mix(in srgb, var(--green-light) 80%, transparent)
    This is native CSS, not a SCSS function — Vite passes it through correctly.

Feature: Feat-006 | Accordion open/close animation
----------------------------------------------------
What was implemented:
  .faq__answer-wrapper uses display: grid with grid-template-rows: 0fr (closed)
  transitioning to 1fr (open). Inner .faq__answer has overflow: hidden and
  min-height: 0. Transition: grid-template-rows 0.3s ease on the wrapper.
  This animates height from 0 to auto without any JS height measurement.
Files changed:
  vibe-coding/src/website/pages/homePage/faq/faq.scss
Engine components considered:
  - None.
Note: grid-template-rows 0fr/1fr is the most reliable cross-browser technique
for animating height-to-auto. max-height approach avoided — it requires an
arbitrary ceiling value and produces sluggish ease-out. display: none/block
avoided — it cannot be CSS-transitioned.
Tradeoffs / follow-up:

Amendment: Amend-001 | FAQ pill — border and box-shadow wrap entire open pill
-------------------------------------------------------------------------------
What was changed:
  Originally the border (2px solid var(--black)) and box-shadow
  (4px 4px 0px 0px var(--sub-text-color)) were on .faq__question, meaning only
  the question button row had the outline. When a pill opened the answer appeared
  below with no border, making the pill look split.
  Moved both border and box-shadow to .faq__pill so they wrap question + answer
  as one unified container. Re-enabled overflow: hidden on .faq__pill (it had
  been removed to avoid clipping) — the grid-template-rows animation does not
  require the pill itself to have overflow: visible; only the inner
  .faq__answer needs overflow: hidden for the 0fr collapse to work.
  Removed border and border-radius from .faq__question (set border: none).
Files changed:
  vibe-coding/src/website/pages/homePage/faq/faq.scss
Engine components considered:
  - None.
Tradeoffs / follow-up:
  overflow: hidden on .faq__pill clips the answer to the pill's border-radius,
  which is the desired visual result. The grid row animation is unaffected.
