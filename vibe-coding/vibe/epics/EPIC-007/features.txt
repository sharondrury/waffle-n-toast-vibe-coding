Feat-004 | contactValidation.js
---------------------------------
File: vibe-coding/src/website/pages/homePage/contactSection/contactValidation.js

Pure JS module — no React, no JSX. Exports a single validate function.

  export function validateContactForm({ name, companyName, email, message }) {
    const errors = {}
    if (!name.trim()) errors.name = 'Name is required.'
    if (!email.trim()) errors.email = 'Email is required.'
    else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email))
      errors.email = 'Please enter a valid email address.'
    if (!message.trim()) errors.message = 'Message is required.'
    // companyName is optional — no validation needed
    return errors  // empty object = valid
  }

Rules:
  - name: required
  - companyName: optional
  - email: required + valid format
  - message: required
  - Returns an errors object — empty means valid, keys match field names

Feat-005 | contactForm.jsx
----------------------------
File: vibe-coding/src/website/pages/homePage/contactSection/contactForm.jsx

Self-contained form component used in both the inline contact section
and the Get In Touch modal. Props:
  - onSuccess (function) — called after successful EmailJS send
  - onError   (function) — called after failed send (optional, can show inline)

Internal state:
  const [fields, setFields] = useState({ name: '', companyName: '', email: '', message: '' })
  const [errors, setErrors] = useState({})
  const [sending, setSending] = useState(false)
  const [submitStatus, setSubmitStatus] = useState('idle')
  // submitStatus values: 'idle' | 'success' | 'error'

Fields rendered (left column, stacked):
  1. Name           — text input, required, label "Name"
  2. Company Name   — text input, optional, label "Company Name (optional)"
  3. Email          — email input, required, label "Email"
  4. Message        — textarea, required, label "Message", min-height 8rem

Each field:
  - Label above input
  - Input/textarea with controlled value and onChange
  - Inline error message below input when errors[field] exists
  - Error input: border colour var(--red) or similar error token

Submit button:
  - Label: "Send Message"
  - Background var(--green-dark), colour var(--white)
  - Padding 0.875rem 2.5rem, border-radius 0.5rem
  - Disabled + "Sending..." while sending === true
  - Full width of the left column

EmailJS send call:
  emailjs.send(
    'service_kg75p6l',
    'template_hkes3vq',
    {
      from_name:    fields.name,
      company_name: fields.companyName,
      reply_to:     fields.email,
      message:      fields.message,
    },
    import.meta.env.VITE_EMAILJS_PUBLIC_KEY
  )

On success:
  setSubmitStatus('success')
  call onSuccess() if provided

On error:
  setSubmitStatus('error')
  call onError() if provided

Success state render:
  Hide form fields and submit button.
  Show message: "Message sent — we'll be in touch!"
  Show a "Send another message" button that calls:
    setSubmitStatus('idle')
    setFields({ name: '', companyName: '', email: '', message: '' })
    setErrors({})
  This resets the form without a page refresh.

Error state render:
  Keep form visible.
  Show inline message: "Something went wrong — please try again."
  Styled in var(--red) or var(--bottom-color).

Validation on submit:
  Call validateContactForm(fields) from contactValidation.js
  If errors returned, set errors state and return early — do not send.
  Clear individual field error on that field's onChange.

Feat-007 | Get In Touch Modal
-------------------------------
The phone icon button in navBar.jsx currently has an animation (EPIC-001).

New files:
  vibe-coding/src/website/components/getInTouchModal/getInTouchModal.jsx
  vibe-coding/src/website/components/getInTouchModal/getInTouchModal.scss

Modal content mirrors the contactSection right column info + contactForm,
but displayed in a modal overlay rather than inline on the page.

Modal structure:
  Backdrop (.modal-backdrop):
    - Position fixed, inset 0, background rgba(0,0,0,0.6), z-index 200
    - Click backdrop to close
  Modal card (.get-in-touch-modal):
    - Background: var(--grey-bg)
    - Border-radius: 1.25rem (20px)
    - Padding: 2rem
    - Max-width: 36rem, width: 90%
    - Position relative
    - Max-height: 90vh, overflow-y: auto (for smaller screens)
  Close button (.get-in-touch-modal__close):
    - Position absolute, top 1rem, right 1rem
    - Background none, border none, font-size 1.5rem, cursor pointer
    - Colour: var(--sub-text-color)
  Heading: "Get in touch" — var(--green-dark), font-size 1.5rem, font-weight 700
  Subheading paragraph — same copy as contactSection right column
  ContactForm rendered below subheading
    - onSuccess closes the modal after a short delay (setTimeout 1500ms)

Modal state lives in navBar.jsx or a parent that can pass isOpen + onClose props.

Recommended approach — modal state in App.jsx or homePage.jsx:
  const [modalOpen, setModalOpen] = useState(false)
  Pass setModalOpen(true) down to NavBar as a prop: onPhoneClick
  NavBar phone button onClick calls onPhoneClick()
  GetInTouchModal rendered at App or homePage level, receives
    isOpen={modalOpen} onClose={() => setModalOpen(false)}

Alternative simpler approach — modal state inside navBar.jsx:
  const [modalOpen, setModalOpen] = useState(false)
  Render GetInTouchModal directly inside the NavBar return, conditionally
  This avoids prop drilling but places a modal inside the nav — acceptable
  for this project scale.
  Claude Code should choose whichever fits the existing navBar.jsx structure.
