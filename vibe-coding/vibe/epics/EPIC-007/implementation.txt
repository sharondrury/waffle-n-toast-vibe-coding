Feature: Feat-005 | contactForm.jsx
--------------------------------------
What was implemented:
  Controlled form with 4 fields (name, companyName, email, message).
  State: fields, errors, sending, submitStatus ('idle' | 'success' | 'error').
  Inline validation via validateContactForm on submit.
  EmailJS send: service_kg75p6l / template_hkes3vq / VITE_EMAILJS_PUBLIC_KEY.
  Success: hides form, shows confirmation + "Send another message" reset button.
  Error: keeps form visible, shows inline retry message.
  Props: onSuccess, onError — both optional, called after send resolves.
  @emailjs/browser installed via npm (was not present from EPIC-004).
Files changed:
  src/website/pages/homePage/contactSection/contactForm.jsx
  src/website/pages/homePage/contactSection/contactForm.scss
Engine components considered:
  - websiteContext.jsx — not used. Form state is local, no need to persist.
  - toast.jsx — not used. Success/error feedback is inline within the form.

Learning note:
  contactForm.jsx is intentionally prop-driven (onSuccess, onError) so it
  can be reused in both the inline contactSection and the getInTouchModal
  without duplicating any logic. One component, two consumers.

Feature: Feat-006 | contactSection.jsx
-----------------------------------------
What was implemented:
  forwardRef wrapper so homePage.jsx can attach a ref for scroll targeting.
  Two-column layout: 60% form left, 40% info right.
  Background var(--grey-bg) on the inner container, not the section wrapper —
  keeps the section flush with the page background.
  Right column uses image2.png from assets/images/ (no contact-specific image
  was available in the asset folder).
  ContactSection.displayName set for React DevTools clarity.
Files changed:
  src/website/pages/homePage/contactSection/contactSection.jsx
  src/website/pages/homePage/contactSection/contactSection.scss
  src/website/pages/homePage/homePage.jsx (import + render with ref)
Engine components considered:
  - websiteContext.jsx — not needed. No persistent state in this section.

Learning note:
  forwardRef lets a parent component (homePage.jsx) attach a ref directly
  to a child component's DOM element. Without it, ref={contactRef} on
  <ContactSection> would not reach the underlying <section> tag, and
  scrollIntoView would fail silently.

Feature: Feat-007 | Get In Touch Modal
-----------------------------------------
What was implemented:
  Modal state (modalOpen) lives inside navBar.jsx — simplest approach,
  avoids prop drilling through App → homePage → NavBar.
  Phone button onClick opens modal; backdrop click and × button close it.
  e.stopPropagation() on modal card prevents backdrop close on inner clicks.
  onSuccess callback auto-closes after 1500ms (setTimeout).

  Modal structure split into two zones:
    .get-in-touch-modal__header — fixed, never scrolls (heading + subheading)
    .get-in-touch-modal__body   — overflow-y: auto, scrolls independently
  The modal card itself uses display: flex / flex-direction: column /
  overflow: hidden — the card stays at a fixed max-height (90vh) while only
  the body area scrolls. Scrollbar appears inside the form area only.

  SCSS path correction: getInTouchModal.scss is at
  src/website/components/getInTouchModal/ — requires ../../../assets/scss/ui/
  (three levels up), not ../../ (two levels).
Files changed:
  src/website/components/getInTouchModal/getInTouchModal.jsx
  src/website/components/getInTouchModal/getInTouchModal.scss
  src/website/components/navBar/navBar.jsx (useState, modal import, button wiring)
Engine components considered:
  - modal.jsx — reviewed; not used. Engine modal does not support the
    fixed-header / scrollable-body structure needed here.

Feature: Feat-008 | Hero CTA Scroll
--------------------------------------
What was implemented:
  hero.jsx "Get your quote" changed from <a href="#contact"> to a <button>
  accepting an onCtaClick prop.
  homePage.jsx creates contactRef = useRef(null), passes:
    <Hero onCtaClick={() => contactRef.current?.scrollIntoView({ behavior: 'smooth' })} />
    <ContactSection ref={contactRef} />
  html { scroll-behavior: smooth } added to index.scss as a global rule.
Files changed:
  src/website/pages/homePage/hero/hero.jsx
  src/website/pages/homePage/homePage.jsx
  src/index.scss
Engine components considered:
  - None. Pure DOM ref + scrollIntoView.

Learning note:
  scrollIntoView({ behavior: 'smooth' }) is the preferred approach over
  hash links (#contact) with React Router — it doesn't alter the URL bar
  or cause route re-evaluation. The ?. optional chaining guards against the
  rare case where the ref hasn't attached yet on first render.
