Feature: Feat-002 | Footer File Setup + App.jsx Integration
--------------------------------------------------------------
What was implemented:
  Created footer.jsx and footer.scss in src/website/components/footer/.
  Imported Footer into App.jsx and rendered it below <AppRoutes /> inside a

  Added padding-bottom: var(--footer-height) to body in index.scss so the
  fixed footer does not hide page content.

Learning note:
  Rendering Footer in App.jsx outside <Routes> is the correct pattern for
  shared layout elements. It renders once regardless of the active route.
  No need to import Footer into every page component.

Feature: Feat-004 | Column One — Logo, Address, Email
-------------------------------------------------------
What was implemented:
  WnT-logo.svg imported as a URL via standard ES import (Vite default for SVG).
  Rendered as <img src={wntLogo} alt="Waffle n Toast">. Build confirmed Vite
  processed and hashed it (WnT-logo-CI1-WjqO.svg in dist/assets/).
  No ?react suffix needed — the logo rendered correctly as <img>.
  <address> element used for semantic HTML. font-style: normal overrides the
  browser default italic on <address>.
  Placeholder address and email used — developer should update before launch.
Files changed:
  src/website/components/footer/footer.jsx
  src/website/components/footer/footer.scss
Engine components considered:
  - None.

Learning note:
  Vite handles SVG imports as URL strings by default when you do:
    import wntLogo from './file.svg'
  If you need to style SVG internals (fill, stroke via CSS), use the ?react
  suffix to import as a React component instead:
    import WntLogo from './file.svg?react'
  For a logo used as a static image, the URL import is simpler and sufficient.

Feature: Feat-006 | Column Three — Social Icons
-------------------------------------------------
What was implemented:
  Three inline SVG icon components defined above the Footer function:
    FacebookIcon  — fill="currentColor" (inherits color from parent)
    InstagramIcon — stroke="currentColor" with rounded rect + inner circle
    LinkedInIcon  — fill="currentColor"
  All icons 20x20px. No external icon library — zero additional dependency.
  All social links: target="_blank" rel="noopener noreferrer".
  Placeholder URLs used — developer must update to real profile URLs.
  Column heading "Follow" matches "Links" heading style.
  Hover colour: var(--right-color) on all links and social icons.
Files changed:
  src/website/components/footer/footer.jsx
  src/website/components/footer/footer.scss
Engine components considered:
  - None.

Learning note:
  Using fill="currentColor" and stroke="currentColor" on SVGs means the icon
  colour is controlled entirely by CSS (the color property on the parent element).
  This lets a single hover rule on .footer__social-link change both the text
  label and the SVG icon colour simultaneously — no separate SVG fill override needed.
