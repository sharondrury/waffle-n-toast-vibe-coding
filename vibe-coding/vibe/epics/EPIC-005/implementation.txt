EPIC 005 — Showcase Carousel & Modal: Implementation Log
=========================================================

Implementation notes for Claude Code:
  - Duplicate array modulo (Feat-008): The carousel renders thumbnailImage twice
    back-to-back. Clicking card at index i in the doubled array must map back
    to the correct modal entry using:
      realIndex = clickedIndex % thumbnailImage.length
    Without this, clicking any card in the second copy would be out of bounds or
    show the wrong modal content.
  - Drag + animation-play-state (Feat-006): During drag, pause the CSS animation
    by setting trackRef.current.style.animationPlayState = 'paused' and apply the
    drag delta as an inline transform on the ref. On mouse-up, clear both inline
    styles (set to '') so the CSS class takes back control. DO NOT use useState
    for drag position — updating state on every mousemove causes re-renders and
    jank. Use refs only (isDragging, startX, dragMoved) for all drag tracking.
  - e.stopPropagation() on modal content (Feat-010): The backdrop div has
    onClick = onClose. The content div inside it MUST call e.stopPropagation()
    on its onClick. Without it, any click inside the modal — images, heading,
    close button area — bubbles up to the backdrop and fires onClose immediately.
  - Seamless marquee loop: Use margin-right on each card instead of CSS gap on
    the track container. With N cards each (cardWidth + marginRight)px wide,
    translateX(-50%) lands exactly at the start of the second copy. CSS gap
    gives N items N-1 gaps, leaving a fractional offset that causes a visible
    jump on loop.
  - Drag click suppression: set dragMoved ref (false on mousedown, true when
    delta > 5px). Check in handleCardClick and return early if true. This
    prevents a drag-release from accidentally opening the modal.

Feature: Feat-006 | Carousel — pause, hover and drag interactions
------------------------------------------------------------------
What was implemented:
  Hover: isPaused state toggles .showcase__track--paused CSS class.
  Modal: isModalOpen also applies the class. Combined: isAnimPaused = isPaused || isModalOpen.
  Drag: all tracking in refs (isDragging, startX, dragMoved). Mouse and touch
  handlers registered on window inside mousedown/touchstart, removed on release.
  During drag: inline animationPlayState = 'paused' + inline transform.
  On release: both cleared (set to ''), CSS class resumes control.
  dragMoved threshold 5px suppresses accidental card click on drag-release.
Files changed:
  vibe-coding/src/website/pages/homePage/showcase/showcase.jsx

Feature: Feat-010 | Modal — close behaviour
---------------------------------------------
What was implemented:
  Backdrop div onClick={onClose}. Content div onClick={e => e.stopPropagation()}.
  stopPropagation is critical — without it clicks inside the modal bubble to the
  backdrop and fire onClose.
  x button calls onClose directly.
  Transition: @keyframes fadeIn (opacity 0->1) on backdrop,
  @keyframes popIn (opacity+scale 0.95->1) on content box.
