  Status: done

Feat-005 | Carousel — auto-scroll animation (right to left, seamless loop)
  - Cards translate continuously from right to left using CSS animation or
    a requestAnimationFrame loop driven by a ref
  - Recommended approach: CSS @keyframes marquee on a flex track container
      translateX(0) → translateX(-50%) when the array is doubled, the second
      half is a perfect copy so the reset is invisible to the user
  - Animation plays on mount, is smooth and has no visible jump on loop
  - Speed: consistent, comfortable reading pace (suggest ~30s–40s for full cycle)
  Status: done

Feat-006 | Carousel — pause, hover and drag interactions
  - Pause on hover:
      When the user's cursor enters any card or the carousel track,
      the auto-scroll animation pauses (animation-play-state: paused)
  - Resume on mouse leave:
      When the cursor leaves the carousel area, auto-scroll resumes
      from the current position (not reset to start)
  - Drag left/right with cursor:
      Mouse down on carousel → record startX position
      Mouse move → translate the track by the delta from startX
      Mouse up → release drag, resume auto-scroll from new position
      Drag should feel physical — no snapping, no momentum required
  - Touch support (mobile): same logic using touchstart / touchmove / touchend
  Status: done

Feat-007 | Carousel — pause when modal is open, resume on close
  - showcase.jsx manages a boolean state: isModalOpen
  - isModalOpen is passed down to the carousel track
  - When isModalOpen is true, auto-scroll pauses regardless of hover state
  - When isModalOpen returns to false, auto-scroll resumes
  - This state is also passed to modalShowcase.jsx to drive open/close
  Status: done

Feat-008 | Modal — open on card click
  - Clicking a carousel card opens the modal
  - The index of the clicked card is stored in state (selectedIndex)
  - selectedIndex is passed to modalShowcase.jsx to determine which
    modalShowcaseContent object to display
  - Modal is rendered in showcase.jsx and conditionally shown based on
    isModalOpen state
  Status: done

Feat-009 | modalShowcase.jsx — modal layout and content
  - Uses the modalShowcaseContent object at selectedIndex
  - Layout:
      Heading: at the top of the modal, from the heading key
      Three images: side by side in a flex row, 8px gap between each
        Each image uses image1, image2, image3 keys as src values
      × close button: top-right corner of the modal
  - Images are placeholder strings until real assets are provided —
    render as <img src={item.image1} alt="" /> etc.
  - Modal sits above the carousel (z-index) with a backdrop overlay behind it
  Status: done

Feat-010 | Modal — close behaviour
  - Close via × button (top-right)
  - Close via clicking the backdrop overlay outside the modal content box
  - Both call the same onClose handler passed from showcase.jsx
  - Closing sets isModalOpen to false and selectedIndex to null
  - Smooth open/close transition recommended (opacity + scale or translateY)
  Status: done

Feat-011 | Carousel — drag-to-scroll (user-controlled horizontal scroll)
  - The user can click/press and hold anywhere on the carousel track and
    drag left or right to scroll the cards manually
  - Behaviour:
      Mouse down / touchstart → record startX, pause auto-scroll
      Mouse move / touchmove  → translate the track by (currentX - startX) delta
      Mouse up / touchend     → release drag, auto-scroll resumes from current position
  - Drag should feel physical and direct — no snapping, no momentum required
  - A drag of more than 5px suppresses the card-click/modal-open so a drag-release
    does not accidentally open a modal
  - Works on both desktop (mouse) and mobile (touch)
  - All tracking held in refs (isDragging, startX, dragMoved) — NOT in state —
    to avoid re-render jank on every move event
  Note: The drag mechanism shares the pause/resume infrastructure already built in
  Feat-006. Feat-011 documents and verifies drag-to-scroll as a standalone
  user-facing capability, separate from hover-pause behaviour.
  Status: done — implemented as part of Feat-006. No additional code needed.
